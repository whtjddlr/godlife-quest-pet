# Calm Pixel — 갓생 퀘스트 펫 디자인 시스템

> **v3 — Pixel Cozy (현행)**: 게임보이/다마고치 감성 전면. 따뜻한 세피아
> 팔레트(#f2e7d0 배경, #463c28 잉크, GB 리프 그린 #7aa23e), 픽셀 폰트
> (DungGeunMo) 타이틀·숫자, 청키 2px 잉크 보더 + 하드 섀도의 "게임 버튼"
> (누르면 3px 물리적으로 눌림), 베젤을 두른 다마고치 디바이스 스크린,
> 게임 데크 탭바. 구현은 styles.css 말미의 PIXEL COZY 단일 레이어
> (제로베이스 리라이트, 이전 30개 레이어 폐기, 픽셀 아트 원본 보존).
>
> 이하 v1/v2 기록은 역사적 참조.

> 컨셉: **픽셀 펫이 유일한 주인공, UI는 차분한 무대.**
> 사용자가 기억할 것은 펫과 오늘의 미션 하나. 나머지 UI는 조용히 물러난다.

## 1. 핵심 원칙

1. **주인공은 픽셀 아트다** — 펫·소품·네임플레이트만 도톰하고 알록달록할 수 있다. UI는 그 대비를 만들기 위해 존재한다.
2. **같은 정보를 두 번 말하지 않는다** — 창 제목과 패널 제목, 진행률과 카운트, 보상 텍스트와 보상 칩 중 하나만 남긴다.
3. **읽는 라벨보다 보는 아이콘** — 지표(컨디션/코인/XP/우정)는 의미색 아이콘+숫자. 단, **동작 버튼은 아이콘만 두지 않는다** (아이콘+라벨).
4. **행동 요소와 정보 요소를 같은 급에 두지 않는다** — 읽기 전용 상태는 HUD·칩으로, 카드 자리는 행동(인증, 밥주기)에게.
5. **화면 크기는 흔들리지 않는다** — 풀스크린 창은 항상 같은 높이, 변하는 숫자는 tabular numerals, 제목은 1줄 말줄임.

## 3. 표면·간격·타이포

- **그림자 2겹 원칙**: 접촉 `0 1-2px` + 원거리 `0 10-16px` 저투명. 크고 흐린 단일 그림자 금지.
- **라운드**: 주요 카드 16px · 행 카드 14px · 컨트롤 12px · 칩 999px.
- **행 간격**: 패널 그리드 gap 8~10px. 개별 margin 대신 부모 gap.
- **타이포 위계**: 창 제목 18/750 · 섹션 제목 17/700 · 본문 14 · 보조 11/500. 굵기 800+는 숫자와 CTA에만.
- 본문 Pretendard Variable(자간 -0.01em), 픽셀 액센트는 DungGeunMo(펫 이름판 전용).

## 4. 아이콘 문법

- **스타일**: 통통한 필 + 라운드 (24px 그리드, 컷아웃은 evenodd). 가는 아웃라인 금지.
- **구현**: CSS 마스크(`data-icon` → `--*-icon` 변수). 색은 background로 주입.
- **세트**: 탭 5(하우스/클립보드체크/유저즈/스파클/쇼핑백) · 헤더 4(다운로드/벨/슬라이더/로그아웃) · 퀘스트 카테고리 7 · 지표 5(rw-*) · 케어 2(칵테일/게임패드) · 펫 행동 5(발자국/매트/플레이/벤치/운동화).
- 동작 버튼에 아이콘을 쓸 땐 반드시 짧은 라벨 동반. 지표는 아이콘+숫자만.

## 5. 화면 구조

### 홈 (4구역)
`헤더(아바타+알림+설정) → 펫 룸(+HUD ❤🔥🪙⚡) → 케어 1행(밥주기 1.5fr + 유흥/게임) + 미션 카드 → 탭바`
- 출석 보상은 접속 시 자동 수령(버튼 없음). 갱신 타이머는 미션 카드 아래 한 줄.
- 시간대별 하늘(새벽~밤)은 유지하되 저채도 팔레트 사용.

### 풀스크린 창 (미션/그룹/소식/상점)
- 항상 전체 높이, 내용은 내부 스크롤. 창 제목 하나만(패널 제목 숨김).
- 진입: 240ms 슬라이드-스케일. 탭 전환: 패널 240ms 페이드-업.

### 미션
카테고리 배너(열림=accent-soft) → 퀘스트 카드(아이콘 · 제목 · 보상 칩 ⚡🪙❤ · 설명 1줄 · 완료 요청(민트)/보기(화이트)).

### 그룹
확인할 인증(최우선) → 방 카드(코드=accent-soft 칩) → **그룹 보상 3행 컴팩트**(카드당 1행: 제목+진행바+남음 칩) → 그룹원 카드(중립 게이지+민트 채움).

### 상점
우정 보상 → 성장 보상(2열 컴팩트 셀) → 캡슐/펫알(내용 아래 풀폭 민트 버튼, 확률은 등급색 칩) → 도감/보관함.

## 6. 모션 언어

- **하나의 커브**: `cubic-bezier(0.2, 0.7, 0.3, 1)` — 창 240ms, 패널 240ms, 모달 200ms.
- **프레스 피드백 공통**: `translateY(1px) scale(0.97)`.
- **펫 안무 원칙**: 예비동작 → 본동작 → 착지 반동(스쿼시&스트레치). 포즈 제거는 애니메이션 종료 후(1.9s).
  산책=깡총 왕복 · 운동=점프 스쿼트 3회 · 런닝=잰걸음 · 영상휴식=뒤로 기대 까딱 · 축하=기쁨 점프.
- 전부 `prefers-reduced-motion` 존중.

## 7. 접근성 규칙

- 시각적으로 숨긴 라벨은 DOM에 유지(font-size 0) — 스크린리더가 읽는다.
- 아이콘 단독 표시는 `role="img"` + `aria-label`.
- `:focus-visible` 민트 아웃라인, 입력 포커스 링 `rgba(53,163,129,.18)`.
- `[hidden]`은 전역 `display:none !important` (컴포넌트 display 규칙이 이기지 못하게).

## 8. 유지보수 노트

- 스타일은 `styles.css` 말미의 Calm Pixel 레이어들에 append-only로 쌓는다. 기존 레이어 수정 대신 아래에 오버라이드.
- 새 지표·카테고리 추가 시: JS는 키만 반환, 아이콘은 CSS `data-icon` 한 줄.
- 배포 후 구버전 화면이 보고되면 `sw.js`의 `CACHE_NAME` 버전을 올린다.
